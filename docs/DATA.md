# Data dictionary

Inputs consumed by the Nomad ABM. Until the Zenodo archive lands
(Phase 9 of the publication-readiness pass), these files live under
`Data/` in the repository. The shipped layout is documented here so a
reviewer can identify, locate, and trace every input used by the
model.

## Coordinate system

All spatial layers are projected in **ITM (Israel Transverse Mercator,
EPSG:2039)**. Distance units are **metres**; cell side length is
**250 m**.

## Spatial extent

Common grid for all rasters: **318 columns × 280 rows = 89 040 cells**,
covering approximately **3 251.54 km²** of the Negev Highlands. The
binary `place_raster` further restricts valid agent activity to the
Israeli portion of the highlands where environmental data coverage is
complete.

## Files

| File | Format | Loaded by | Contents | Source / derivation |
|---|---|---|---|---|
| `yearly_data.h5` | HDF5 | `LazyYearlyData` in `nomad_abm.model` | Per-year stack of annual rasters: `annual_rainfall`, `veg_ras`, `ag_ras`, `arable_soil_distance`. Each group keyed `group_<i>` corresponds to one historical IMS rainfall year. | Generated by `convert_to_numpy.ipynb` (legacy notebook; see `git log -- legacy/`) from raw IMS station records via IDW interpolation. |
| `yearly_data_10_25.h5` | HDF5 | same | Subset of `yearly_data.h5` containing only the years used in the calibration run window. Used by default for performance. | Same source; subsetted offline. |
| `per_data.h5` | HDF5 | `nomad_abm.model.load_permanent_data` | Permanent rasters: `mean_annual_rainfall`, `slope`, `kb_distance` (cost-distance to Tel el-Qudeirat), `pw_distance` (cost-distance to permanent water). | IMS (rainfall, IDW), JAXA ALOS PALSAR DEM (slope, 12.5 m resampled to 250 m), Tobler cost-distance from a single point (Tel el-Qudeirat), Tobler cost-distance from springs/wells layer. |
| `per_data_10_25.h5` | HDF5 | same | Subset matching `yearly_data_10_25.h5`. | Same source. |
| `ext_raster.npy` | NumPy 2-D array (`float32`, 318×280) | `nomad_abm.model` (`GlobalData.ext_raster`) | Suitability/accessibility raster used as a model-wide context layer. | Derived from a GeoTIFF input; details TBD. |
| `place_raster.npy` | NumPy 2-D array (`uint8`, 318×280) | `nomad_abm.model` (`GlobalData.place_raster`) | Binary mask. 1 = valid placement cell, 0 = excluded (outside Israel or outside the highlands). | Manually digitised over modern Israel-Egypt border and topographic boundary. |
| `distac_pw.tif` + `distac_pw.tfw` | GeoTIFF | (raw input for `pw_distance` reproduction) | Cost-distance from permanent water sources, in walking hours. | ArcGIS Pro Distance Accumulation tool with Tobler's hiking function over the DEM. |
| `yearly_outputs.pkl` | pickle | downstream analysis scripts | Cached per-year processed numpy arrays from earlier runs; not required for a fresh simulation. | Generated by an earlier notebook run. |
| `cntrl_outputs.pkl` / `cntrl_outputs.pbz2` | pickle (raw / bz2) | analysis scripts | Reference outputs for control comparisons. | Generated by an earlier notebook run. |
| `P_for_calib.shp` (+ `.dbf` / `.shx` / `.prj`) | ESRI shapefile | `nomad_abm.model.objective_function` | 462 archaeological sites within the model boundary. Attribute `value`: `1 = other architectural site`, `2 = enclosed compound`. | Compiled from Israel Antiquities Authority survey records (Chapter 3) and legacy surveys (Glueck 1958; Aharoni et al. 1960; Rothenberg 1967). **Currently missing from the repository; must be placed at `Data/P_for_calib.shp` or pointed to via `NOMAD_ABM_CALIB_SHP`.** |

## Field reference inside the HDF5 files

`yearly_data.h5` and `yearly_data_10_25.h5`:

```
/metadata
  attrs:
    num_groups : int              # number of historical rainfall years
/group_<i>                        # one group per year, i = 0..num_groups-1
  /annual_rainfall   : float32 [318, 280]   # mm, normalized 0–10
  /veg_ras           : float32 [318, 280]   # 0–10
  /ag_ras            : float32 [318, 280]   # 0–10
  /arable_soil_distance : float32 [318, 280]   # hours walking
```

> The exact dataset names and dtypes inside each group should be
> confirmed against `h5py.File(path).visit(print)` on the data
> machine; the above reflects the code's expectations as of
> `nomad_abm.model.LazyYearlyData.__getitem__`.

`per_data.h5` follows the same group/dataset convention with one
group (the permanent layers).

## Zenodo plan (Phase 9)

When Phase 9 lands, all files listed above plus the missing
`P_for_calib.shp` will be bundled into a versioned `.zip`, uploaded to
Zenodo with a DOI, and fetched from the repo via
`scripts/download_data.py` with SHA-256 checksums. The repository
itself will then carry only a tiny sample fixture under
`tests/fixtures/`.
